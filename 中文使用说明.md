## 1. 首次安装

本项目提供两套安装脚本，请根据您的需求选择。

### 方式 A：Web UI 版（推荐）

Web UI 版提供一个类似 ChatGPT 的浏览器界面，可以实时查看 Agent 的思考过程、管理工作区文件，体验更好。

| 操作系统 | 运行脚本 |
|---|---|
| **Windows** | `install_webui_windows.ps1` |
| **macOS / Linux** | `install_webui_unix.sh` |

安装完成后，文件会被放置在 `E:\OpenManus\OpenManusWeb` 文件夹中（Windows）或用户目录下的 `OpenManusWeb` 文件夹中（macOS/Linux）。

### 方式 B：终端版（经典）

终端版是传统的命令行交互方式，轻量快速。

| 操作系统 | 运行脚本 |
|---|---|
| **Windows** | `install_windows.ps1` |
| **macOS / Linux** | `install_unix.sh` |

安装完成后，文件会被放置在 `E:\OpenManus` 文件夹中（Windows）或用户目录下的 `OpenManus` 文件夹中（macOS/Linux）。

### 安装步骤（两种方式通用）

1.  **下载仓库文件**：将本项目所有文件下载到您的电脑。
2.  **运行脚本**：
    -   **Windows**: 右键点击对应的 `.ps1` 脚本 → 选择 "**使用 PowerShell 运行**"。
    -   **macOS/Linux**: 打开终端，进入项目目录，执行 `chmod +x <脚本名>.sh && ./<脚本名>.sh`。
3.  **输入 API Key**：脚本会提示您输入 OpenAI API Key。请前往 [platform.openai.com/api-keys](https://platform.openai.com/api-keys) 创建一个。**强烈建议不要跳过此步**。
4.  **选择默认模型**：脚本会让您选择默认使用的模型，推荐选择 GPT-4.1-mini（最便宜）。

> **提示**：如果 Windows 提示"无法加载文件...因为在此系统上禁止运行脚本"，请打开一个新的 PowerShell 窗口，输入 `Set-ExecutionPolicy Bypass -Scope Process` 后再试。

> **注意**：两种版本互不影响，可以同时安装。

## 2. 日常使用

### Web UI 界面（推荐）

这是最推荐的使用方式，您可以在浏览器中与 Agent 进行类似 ChatGPT 的交互。

| 操作系统 | 启动方式 |
|---|---|
| **Windows** | 双击 `启动WebUI.bat` 文件 |
| **macOS / Linux** | 在终端执行 `./start_webui.sh` |

启动后，脚本会自动打开一个本地服务器。请在浏览器中访问 [http://localhost:8000](http://localhost:8000)，即可看到一个美观的聊天界面。

**Web UI 核心功能**：

| 功能 | 说明 |
|---|---|
| **聊天对话** | 像 ChatGPT 一样与 Agent 交流，输入任务即可执行 |
| **实时思考过程** | 在界面左侧实时查看 Agent 的每一步思考和操作 |
| **文件浏览器** | 查看和管理 Agent 在工作区生成的文件 |
| **执行进度** | 可视化显示任务执行进度 |

> **提示**：您可以将 `启动WebUI.bat` 的快捷方式放到桌面，方便快速访问。

### 终端模式（经典）

如果您习惯于在终端中操作，也可以使用传统的终端模式。

| 操作系统 | 启动方式 |
|---|---|
| **Windows** | 双击 `启动终端模式.bat` 或 `启动Agent.bat` 文件 |
| **macOS / Linux** | 在终端执行 `./start_terminal.sh` |

启动后，一个终端窗口会打开，您就可以在里面输入您的任务目标，开始与 AI Agent 交互了。

## 3. 切换模型

本方案最大的特色就是可以灵活切换驱动 Agent 的"大脑"（LLM 模型），以平衡性能和成本。

**如何切换**：

| 操作系统 | 操作方式 |
|---|---|
| **Windows** | 双击 `切换模型.bat` 文件 |
| **macOS / Linux** | 在终端执行 `python switch_model.py`（需先激活虚拟环境） |

运行后，您会看到一个清晰的菜单，列出了所有预设模型及其特点。输入数字即可一键切换。

**预设模型对比**：

| 模型 | 单次任务费用 | 推荐场景 |
|---|---|---|
| **GPT-4.1-mini** | **¥0.14** | 日常任务（**推荐默认**） |
| **GPT-5-mini** | ¥0.25 | 需要更强推理时 |
| **GPT-4.1** | ¥0.42 | 全能型，长上下文 |
| **GPT-5.2** | ¥0.86 | 复杂编码/重要任务 |
| **GPT-4.1-nano** | ¥0.07 | 极致省钱，简单任务 |
| **Ollama 本地模型** | **¥0（免费）** | 隐私敏感/离线使用 |

> **建议**：日常使用 GPT-4.1-mini，遇到复杂任务时切换到 GPT-5.2，这样每月花费约 ¥30~100。

## 4. 使用本地 Ollama 模型

如果您的电脑有不错的独立显卡（如 RTX 5070，12GB 显存），可以尝试完全免费、100% 本地运行的方案。

1.  **安装 Ollama**：从 [ollama.com/download](https://ollama.com/download) 下载并安装 Ollama。
2.  **下载模型**：在终端执行以下命令之一：

| 模型 | 下载命令 | 说明 |
|---|---|---|
| **Qwen2.5 14B** | `ollama pull qwen2.5:14b` | 推荐，中文能力强 |
| **DeepSeek-R1 14B** | `ollama pull deepseek-r1:14b` | 推理能力强 |
| **Llama 3.3 8B** | `ollama pull llama3.3:8b` | 轻量级选择 |

3.  **切换模型**：运行模型切换工具，选择对应的 Ollama 模型即可。

> **注意**：本地模型的 Agent 任务成功率（约 40%）低于云端模型（约 90%），建议作为备选方案使用。

## 5. 常见问题 (FAQ)

**Q: OpenAI API Key 在哪里获取？收费吗？**

A: 您可以用您的 ChatGPT Plus 账号登录 [platform.openai.com/api-keys](https://platform.openai.com/api-keys) 创建。API 是独立于 Plus 订阅计费的，但新注册用户有 **$5 免费额度**，使用 GPT-4.1-mini 的话，足够免费用很久了。

**Q: Web UI 版和终端版有什么区别？**

A: 核心 Agent 能力完全一样，区别只在于交互界面。Web UI 版提供浏览器中的聊天界面，可以实时查看思考过程和文件，体验更好。终端版是传统的命令行交互，更轻量。

**Q: 为什么 `config.toml` 文件不见了？**

A: 为了保护您的 API Key 安全，我们已经将 `config/config.toml` 文件添加到了 `.gitignore` 中。这意味着它不会被上传到 GitHub。安装脚本会自动在您的本地目录中创建这个文件。

**Q: 启动时报错怎么办？**

A: 请确保您已经按照安装脚本的指引正确安装了所有依赖。如果问题仍然存在，可以尝试删除对应的安装文件夹（`OpenManus` 或 `OpenManusWeb`），然后重新运行一次安装脚本。

**Q: 我可以自己添加其他模型吗？**

A: 当然可以！运行模型切换工具时，选择"自定义模型"（选项 0），然后输入模型的名称、API Base URL 和 API Key 即可。支持任何兼容 OpenAI API 格式的模型服务。

**Q: Web UI 打不开怎么办？**

A: 请确认终端中没有报错信息。如果端口 8000 被占用，可以手动修改 `web_run.py` 中的端口号。确保防火墙没有阻止本地 8000 端口的访问。
